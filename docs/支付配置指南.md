# 支付接口配置指南

## 概述

本文档指导如何配置 **支付宝当面付** (Barcode Pay) 支付接口。

## 什么是支付宝当面付

当面付是支付宝推出的线下扫码支付解决方案：
- 用户出示付款码 → 商户扫码收款
- 商户生成收款码 → 用户扫码付款

适用于移动 App 会员订阅场景。

## 模式说明

| 模式 | 说明 | 用途 |
|------|------|------|
| `sandbox` | 沙箱环境 | 开发测试，不真实扣款 |
| `production` | 生产环境 | 正式环境，真实扣款 |

## 支付宝当面付配置步骤

### 1. 注册支付宝开放平台账号

访问 [支付宝开放平台](https://open.alipay.com) 注册开发者账号。

### 2. 创建应用

1. 登录开放平台 → 应用中心 → 创建应用
2. 选择「网页/移动应用」
3. 填写应用名称等信息
4. 提交审核 (一般 1-3 工作日)

### 3. 开通当面付产品

审核通过后，在应用详情页：
1. 进入「支付产品」
2. 开通「当面付」(自动开通，无需额外申请)

### 4. 获取配置信息

| 参数 | 获取位置 | 说明 |
|------|---------|------|
| `ALIPAY_APPID` | 应用详情页 | 8位或16位数字 |
| `ALIPAY_PRIVATE_KEY` | 开发设置 → 应用密钥 | 生成 RSA2 私钥 |
| `ALIPAY_PUBLIC_KEY` | 开发设置 → 应用密钥 | 支付宝 RSA2 公钥 |
| `ALIPAY_STORE_ID` | 商户中心 → 门店管理 | 商户门店编号 |

### 5. 配置回调地址

在应用详情页 → 开发设置中设置：
- **授权回调地址**
- **异步通知地址**: `https://yourdomain.com/api/v1/vip/webhook/alipay`

## 环境变量配置

复制 `.env.example` 为 `.env` 并填写配置：

```bash
# 支付模式
PAYMENT_MODE=production

# 支付宝当面付
ALIPAY_APPID=2021000000000000
ALIPAY_PRIVATE_KEY="-----BEGIN RSA PRIVATE KEY-----
MIIEvQIBADANBgkqhkiG9w0BAQEFAASCBKcwggSjAgEAAoIBAQC...
-----END RSA PRIVATE KEY-----"

ALIPAY_PUBLIC_KEY="-----BEGIN PUBLIC KEY-----
MIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8AMIIBCgKCAQEA...
-----END PUBLIC KEY-----"

ALIPAY_STORE_ID=your_store_001

# 回调地址
ALIPAY_NOTIFY_URL=https://yourdomain.com/api/v1/vip/webhook/alipay
```

## 支付流程

```
用户点击购买 VIP
       ↓
API 生成订单 (预下单)
       ↓
返回收款二维码 (qr_code)
       ↓
用户扫码支付 (支付宝 App)
       ↓
支付宝异步通知 (webhook)
       ↓
API 更新订单状态 & 激活 VIP
```

## 沙箱测试

### 1. 开启沙箱

```bash
PAYMENT_MODE=sandbox
```

### 2. 下载沙箱支付宝

- Android: 下载「支付宝沙箱版」
- iOS: 使用 TestFlight 安装沙箱版

### 3. 获取沙箱账号

登录 [沙箱环境](https://openhome.alipay.com/platform/appDaily.htm) 获取：
- 沙箱账号
- 登录密码
- 支付密码

### 4. 测试流程

1. 使用沙箱账号登录沙箱支付宝
2. App 扫码 (沙箱二维码)
3. 输入沙箱支付密码完成支付

## 常见问题

**Q: 当面付和普通支付宝支付有什么区别？**
A: 当面付主要面向线下场景，支持扫码收款和被扫码付款。App 集成推荐使用「预下单生成收款码」方式。

**Q: 审核需要多长时间？**
A: 一般 1-3 个工作日，加急可能当天通过。

**Q: 回调通知收不到？**
A:
- 确认回调地址公网可访问
- 检查防火墙/安全组是否放行
- 验证签名配置是否正确

**Q: 签名验签失败？**
A:
- 检查公私钥是否匹配
- 确认 sign_type 为 RSA2
- 检查编码是否为 UTF-8

## 安全建议

1. **不要提交密钥到 Git**
   - 使用 `.env` 文件
   - 确保 `.env` 在 `.gitignore` 中

2. **定期更换密钥**
   - 建议每 3-6 个月更换
   - 更换后及时更新服务器配置

3. **开启验签**
   - 生产环境务必验证回调签名
   - 防止伪造通知
